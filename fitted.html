Fitted A Responsive Web App

Role: UI Designer
Duration: Dec '22 - Jan '23
Tools: Figma

Overview

Fitted is a responsive web app designed to encourage users to increase their physical activity, even if it’s squeezing in a five minute routine. Users are provided with routines, guides, daily challenges, and interactive examples- viewable on mobile, tablet, and desktop.

Objective

This app allows people to find, save, share, and schedule exercises and routines that suit their preferences, time, and resources available.

Analysis Based on user personas and stories I was provided with, I created user flows in order to understand and visualize how users will interact with the app and what steps they need to take to achieve their goals.

User Stories Provided

  • As a new user, I want to learn about different exercises, so that I can figure out what is best for me.
  • As a new user, I want to be shown how the exercises are done, so that I know I’m doing them correctly.
  • As a frequent user, I want to complete daily challenges, so that I can have an additional way to stay motivated.

User Flow Diagram

It was clear that the main functionalities I would be starting with were the Home, Exercise list, Exercise details page, Quiz, and Daily Challenge. Thus the challenge began to make a content-heavy website feel simple and easy to navigate.

Design I designed with a mobile-first approach in order to focus on the essential content for each screen before having a bigger space to work on.

Low-Fidelity Wireframes

Based on the user stories and user flow diagram, I drew low-fidelity wireframes for all the main functionalities of the web app. I focused on using design elements that new users would find familiar to combat the challenge of being confused using a content-heavy website.



Mid-Fidelity Wireframes

After getting an idea of how I wanted to organize the app’s content, I started adding UI elements, UI design patterns, text, and ideas of the icons I wanted to use, creating mid-fidelity wireframes.

Input feedback

When there is an input error, there is a message of what the error is in red text and the input field is outlined in red.

Empty Slate

An animation or illustration and a message pointing out what the users need to do to fill the screen with content.

Search Autocomplete

Search suggestions appear when users start typing on the search bar.

Modal Window & Sharing

A modal window pops up when users tap the share icon. Users are able to see the exercise they’re sharing and can send it to friends on the app or share via text, email, etc.


Moodboard

At this point, I created two moodboards to figure out what style would suit this app the best.

I think the selected moodboard’s brighter colors and airy/light feel would help users feel more motivated and enjoy learning. The pops of color contrasting the light colors bring attention to the app yet aren’t too distracting. The other moodboard has more neutral, earthy colors and I would use it if I wanted to bring a dark mode to the app.



Visual Style Guide

Since the app is content heavy, the style chosen is rooted in simplicity with pops of color. This is meant to keep user’s engaged - not distracted or bored.

High-Fidelity Wireframes

This is the final design after incorporating various rounds of feedback. The design is light and consistent throughout all screens. I think keeping the style minimal helped counter the challenge I had from the beginning - which was how to make a content-heavy website feel easy to use. The interactions and gestures in the prototype reflect the intuitive and straightforward nature of the web app’s interface.

Responsive Wirefames

As I polished up the mobile designs, I started working on the responsive designs and how to reorganize the mobile layout into a bigger canvas in a cohesive manner.

Tablet - Low-Fidelity to High-Fidelity

I changed how the Exercise list page was formatted to make better use of more screen space. This way users will be able to look through more exercises faster and easier.


I reorganized this screen to hold more information and break it down into sections.


Desktop - Low-Fidelity to High-Fidelity

Desktop has the same list organization as tablet. However, the filter is now on the side so users can leave it open and play around with the filters easily.


Since the screen size allowed for more content, I didn’t want to fill the whole screen with just text. I decided to have the main video and text on the left and the supplementary information on the right side of the screen.

Conclusion

I wanted to create a product that is as easy to use as it is enjoyable. I had the challenge of making an app packed with information feel intuitive and uncomplicated for users. I found that I would get overwhelmed with all the possible design decisions I could make, resulting in a mental block. There are so many ways to organize the layouts of screens so I aimed to focus on making each screen as functional and easy to process as possible. Adding UI patterns helped make each screen feel more familiar for new users.

I struggled with where to position favorite, add, and share buttons so they’re not intrusive or at an awkward spot. Usability testing would’ve been the best thing to do to obtain user feedback and continue the iterative process. I would also incorporate A/B preference tests for small scale design choices that can be quickly changed.
